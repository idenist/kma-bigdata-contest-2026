# 임상도 & 임상지수(FMI) 팀원 설명 스크립트

---

## 1. 이 데이터가 뭐예요? — 한 줄 요약

> **강원도 산림 562,276개 구역에 대해 "이 숲이 불에 얼마나 잘 타는가?"를 수치화한 공간 데이터입니다.**

산림청이 2025년에 발행한 **임상도(1:5,000 축척)** 를 가공한 것으로,  
국가산불위험예보시스템(FFDRI)의 핵심 입력값인 **임상지수(FMI)** 가 포함되어 있습니다.

---

## 2. 핵심 개념 — FFDRI와 FMI의 관계

산불 위험을 숫자 하나로 표현하는 공식이 있습니다.

```
FFDRI = ( 7 × 기상지수(DWI)  +  1.5 × 임상지수(FMI)  +  1.5 × 지형지수(TMI) ) × 일가중치
```

| 지수 | 의미 | 우리 데이터에서 |
|---|---|---|
| DWI (기상지수) | 온도·습도·바람 등 기상 조건 | 기상 관측소 데이터에서 계산 |
| **FMI (임상지수)** | **숲의 수종 구성 → 연소 위험** | **이 파일에서 직접 제공** |
| TMI (지형지수) | 경사·고도·사면 방향 | 별도 지형 데이터 필요 |

**기상 가중치(7)이 가장 크지만**, FMI는 구역별 고정값이라 공간적 위험 차이를 결정하는 핵심입니다.

---

## 3. FMI 값이 어떻게 정해지나요?

수종(나무 종류)별로 **불에 얼마나 잘 타는지**를 산림청이 규정한 값입니다.

| 임상 유형 | 대표 수종 | FMI 값 | 비고 |
|---|---|---|---|
| 침엽수림 | 소나무, 잣나무, 낙엽송 | **10** | 가장 위험 — 수지(油脂) 성분 |
| 혼효림 | 침활 혼합림 | **3** | 중간 |
| 활엽수림 | 참나무류, 신갈나무 등 | **2** | 비교적 안전 |
| 죽림 | 대나무 | **1** | 낮음 |
| 무립목지/비산림 | 제지, 관목덤불 등 | **0** | 분석에서 제외 권장 |

> **핵심 포인트**: 소나무 숲이 활엽수림보다 5배 위험하게 설정되어 있습니다.

## 5. 주요 컬럼 설명

```
polygon_id      → 구역 고유 ID. 다른 데이터와 조인할 때 쓰는 key
FRTP_NM         → 임상명 (침엽수림/활엽수림/혼효림/죽림/무립목지)
FMI             → 임상지수 ★★★ 핵심 컬럼
KOFTR_NM        → 수종명 (소나무, 잣나무 등)
AGCLS_NM        → 영급 (1영급=1~10년생, 5영급=41~50년생)
DMCLS_NM        → 경급 (치수/소경목/중경목/대경목 — 나무 두께)
DNST_NM         → 밀도 (소/중/밀 — 얼마나 빽빽한가)
HEIGHT_NM       → 임분고 (평균 나무 높이)
Shape_Area      → 구역 면적(㎡)
centroid_lon/lat → WGS84 중심 좌표 ★★★ 기상 관측소 매핑에 사용
```

## 6. 가공 과정 요약 (process_forest.py)

원본은 GIS 벡터 파일(.shp)이라 직접 쓸 수 없었습니다. 아래 과정으로 CSV로 변환했습니다.

```
원본 SHP 파일 (강원 서부 51_1 + 강원 동부 51_2)
    ↓
속성 테이블 추출 (DBF 파싱)
    ↓
좌표계 변환: KGD2002(EPSG:5179) → WGS84(EPSG:4326)
    → GIS 전용 좌표계를 일반 경위도로 변환
    ↓
각 구역 중심 좌표(centroid) 계산
    → bbox 중심점 사용 (빠른 계산, 관측소 매핑 정확도 충분)
    ↓
FRTP_NM 기준 FMI 컬럼 추가
    ↓
두 파일 합치기 → forest_fmi.csv (562,276행)
```

## 7. 다른 데이터와 어떻게 결합하나요?

### 기상 데이터와 결합 — "가장 가까운 관측소" 방식

```python
from scipy.spatial import cKDTree

# forest_fmi.csv의 centroid_lat/lon ↔ 기상 관측소 LAT/LON
tree = cKDTree(stations[['LAT','LON']].values)
_, idx = tree.query(forest[['centroid_lat','centroid_lon']].values)
forest['STN_ID'] = stations.iloc[idx]['STN_ID'].values

# 이후 날짜별 기상 데이터를 STN_ID 기준으로 merge
```

### 지형 데이터와 결합 — polygon_id 기준 join

```python
result = forest.merge(terrain_df[['polygon_id', 'TMI']], on='polygon_id')
```

## 8. 주의사항 & 자주 하는 실수

| 주의 | 이유 |
|---|---|
| `FMI == 0` 구역 제외하고 분석 | 무립목지/비산림은 산불 위험 분석 대상이 아님 |
| 좌표는 이미 WGS84로 변환 완료 | KGD2002로 다시 변환하면 안 됨 |
| `polygon_id`는 가공 후 부여된 ID | 원본 SHP의 ID와 다를 수 있음 |
| 갱신년도 NaN 구역 존재 | 오래된 구역일 수 있어 필요 시 필터링 |

## 9. 최종 정리 — 한 장 요약

```
이 파일이 하는 일:
  강원도 56만개 숲 구역 → 각 구역이 "불에 얼마나 잘 타는지(FMI)" 점수화

FFDRI 공식에서 FMI의 역할:
  FFDRI = 7×기상 + 1.5×FMI + 1.5×지형
  → FMI가 높을수록(침엽수림=10), 같은 기상 조건에서도 위험지수 더 높게 나옴

우리 팀이 할 일:
  ① 이 FMI + ② 기상 DWI + ③ 지형 TMI → 구역별 FFDRI 계산 → 예측 모델 입력
```
