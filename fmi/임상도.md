# 임상도 데이터 가공 정리

## 개요

산림청 2025년 임상도(1:5,000) 원본 SHP 파일을 가공하여,  
산불위험지수(FFDRI) 계산에 필요한 **임상지수(FMI)** 를 포함한 CSV로 변환한 작업 폴더입니다.

---

## 파일 설명

### `forest_fmi.csv` — 최종 산출물

강원도 전체 산림 구역 **562,276개**에 대한 속성 테이블입니다.  
원본 SHP의 좌표계(KGD2002/EPSG:5179)를 WGS84(EPSG:4326)로 변환하고,  
임상지수(FMI) 및 중심점 좌표(centroid)를 추가한 파일입니다.

#### 컬럼 설명

| 컬럼명 | 설명 | 비고 |
|---|---|---|
| `polygon_id` | 구역 고유 ID | 다른 데이터와 조인 시 key |
| `STORUNST` | 임목존재코드 | 1=입목지, 2=무립목지, 0=비산림 |
| `FROR_CD` | 임종코드 | 1=인공림, 2=천연림 |
| `FRTP_CD` | 임상코드 | 1=침엽, 2=활엽, 3=혼효, 4=죽림, 0=무립목 |
| `FRTP_NM` | 임상명 | 침엽수림 / 활엽수림 / 혼효림 / 죽림 / 무립목지 |
| **`FMI`** | **임상지수** | **침엽=10, 혼효=3, 활엽=2, 죽림=1, 기타=0** |
| `KOFTR_NM` | 수종명 | 소나무, 잣나무, 낙엽송, 신갈나무 등 |
| `AGCLS_CD` / `AGCLS_NM` | 영급(코드/명) | 1~9영급 (10년 단위, 1영급=1~10년생) |
| `DMCLS_CD` / `DMCLS_NM` | 경급(코드/명) | 치수 / 소경목 / 중경목 / 대경목 |
| `DNST_CD` / `DNST_NM` | 밀도(코드/명) | A=소, B=중, C=밀 |
| `HEIGHT` / `HEIGHT_NM` | 임분고(코드/명) | 나무 평균 높이 (2m 단위) |
| `갱신년도` | 해당 구역 마지막 업데이트 연도 | 없으면 NaN |
| `Shape_Area` | 구역 면적 (㎡) | |
| **`centroid_lon`** | **중심점 경도 (WGS84)** | **기상 관측소 매핑용** |
| **`centroid_lat`** | **중심점 위도 (WGS84)** | **기상 관측소 매핑용** |

#### FMI(임상지수) 값 기준

FFDRI(산불위험지수) 공식에서 임상지수는 아래 값을 사용합니다.

```
FFDRI = { 7×기상지수(DWI) + 1.5×임상지수(FMI) + 1.5×지형지수(TMI) } × 일가중치
```

| 수종 유형 | FMI 값 | 구역 수 |
|---|---|---|
| 침엽수림 (소나무, 잣나무, 낙엽송 등) | **10** | 235,078 (41.8%) |
| 활엽수림 (참나무류, 신갈나무 등) | **2** | 216,491 (38.5%) |
| 혼효림 (침활 혼합) | **3** | 76,625 (13.6%) |
| 죽림 | 1 | 1,012 (0.2%) |
| 무립목지/비산림 | 0 | 33,070 (5.9%) |

> ⚠️ 분석 시 `FMI == 0` (무립목지/비산림) 구역은 제외하는 것을 권장합니다.

#### 좌표 범위

| | 최솟값 | 최댓값 |
|---|---|---|
| 경도(lon) | 127.2001 | 129.3562 |
| 위도(lat) | 37.0283 | 38.3992 |

### `process_forest.py` — 가공 스크립트

원본 SHP → `forest_fmi.csv` 생성을 재현하는 Python 스크립트입니다.

#### 처리 흐름

```
51_1.shp + 51_1.dbf (강원 서부)  ┐
                                  ├→ 속성 추출 + centroid 계산 + WGS84 변환
51_2.shp + 51_2.dbf (강원 동부)  ┘
         ↓
    FMI 컬럼 추가 (FRTP_NM 기준)
         ↓
    두 파일 합치기 (562,276행)
         ↓
    forest_fmi.csv 저장
```

#### 주요 기술 사항

- **좌표 변환**: KGD2002(EPSG:5179) → WGS84(EPSG:4326), `pyproj` 사용
- **중심점**: 각 다각형의 bbox 중심값 사용 (계산 속도 최적화, 관측소 매핑 용도로 충분히 정확)
- **라이브러리**: `dbfread`(DBF 읽기), `pyshp`(SHP geometry), `pyproj`(좌표 변환), `pandas`

#### 실행 방법

```bash
pip install geopandas dbfread pyshp pyproj pandas
python process_forest.py
```

> ⚠️ 원본 SHP 파일 구조 주의: 51_1의 경우 `.shp`/`.shx`는 루트 폴더,  
> `.dbf`/`.prj`는 `01. 2025년 임상도/` 하위 폴더에 위치합니다.  
> 스크립트 실행 전 두 파일을 같은 폴더로 모아두거나, 스크립트 내 경로를 확인하세요.

## 다른 데이터와 결합하는 방법

### 기상 데이터와 결합 (최근접 관측소 방식)

```python
import pandas as pd
from scipy.spatial import cKDTree
import numpy as np

# 임상도 로드
forest = pd.read_csv("forest_fmi.csv", encoding='utf-8-sig')
forest_valid = forest[forest['FMI'] > 0].reset_index(drop=True)  # 무립목지 제외

# 기상 관측소 위치
stations = pd.read_csv("AWS지점정보.csv", encoding='utf-8-sig')
stations = stations[stations['LAW_ADDR'].str.contains('강원', na=False)]

# 최근접 관측소 매핑
tree = cKDTree(stations[['LAT','LON']].values)
_, idx = tree.query(forest_valid[['centroid_lat','centroid_lon']].values)
forest_valid['STN_ID'] = stations.iloc[idx]['STN_ID'].values

# 이후 날짜별 기상값을 STN_ID 기준으로 merge
```

# 예시: 지형지수(TMI) 테이블이 있을 경우
