# 📦 데이터 다운로드 및 적재 방법

본 프로젝트에서 사용하는 원본 데이터는 파일 용량이 크기 때문에 GitHub 저장소에 포함하지 않습니다.  
따라서 아래 안내에 따라 데이터를 직접 다운로드한 후, 데이터베이스에 적재해야 합니다.

---

## 1. 데이터 다운로드

아래 Google Drive 링크에서 데이터 압축 파일을 다운로드합니다.

👉 [데이터 다운로드 링크](https://drive.google.com/file/d/1OJIv3ACHjSg1AHftv4wF6x2909OE1eiN/view?usp=sharing)

---

## 2. 압축 파일 위치

다운로드한 `.zip` 파일을 **이 문서와 동일한 경로**에 위치시킵니다.

예시 구조는 다음과 같습니다.

```text
weather/
├─ DATA_DOWNLOAD.md
├─ 다운로드받은_데이터.zip
├─ db_query/
│  ├─ create_database.py
│  └─ load_data.py
└─ ...
```

---

## 3. 압축 해제

다운로드한 `.zip` 파일의 압축을 해제합니다.

압축 해제 후 데이터 파일들이 프로젝트에서 사용하는 경로에 맞게 위치해야 합니다.

예시:

```text
weather/
├─ data/
│  └─ raw/
│     ├─ 강수량_20191101_20251231.csv
│     ├─ 기상통합_20191101_20251231.csv
│     ├─ 기온_20191101_20251231.csv
│     ├─ 바람_20191101_20251231.csv
│     └─ 습도_20191101_20251231.csv
├─ db_query/
│  ├─ create_database.py
│  └─ load_data.py
└─ DATA_DOWNLOAD.md
```

---

## 4. 데이터베이스 생성

먼저 `db_query` 폴더에 있는 `create_database.py` 파일을 실행합니다.

```bash
python db_query/create_database.py
```

이 스크립트는 프로젝트 실행에 필요한 데이터베이스와 테이블을 생성합니다.

---

## 5. 데이터 적재

데이터베이스 생성이 완료되면, 이어서 `load_data.py` 파일을 실행합니다.

```bash
python db_query/load_data.py
```

이 스크립트는 다운로드한 CSV 데이터를 데이터베이스 테이블에 적재합니다.

---

## 실행 순서 요약

아래 순서대로 진행하면 됩니다.

```bash
# 1. 데이터베이스 및 테이블 생성
python db_query/create_database.py

# 2. CSV 데이터 적재
python db_query/load_data.py
```

---

## 주의사항

- 데이터 파일은 용량이 크기 때문에 GitHub에 업로드하지 않습니다.
- `.csv` 파일은 `.gitignore`에 의해 Git 추적 대상에서 제외됩니다.
- 압축 해제 후 데이터 경로가 맞지 않으면 `load_data.py` 실행 시 파일을 찾지 못할 수 있습니다.
- 반드시 `create_database.py`를 먼저 실행한 뒤 `load_data.py`를 실행해야 합니다.
