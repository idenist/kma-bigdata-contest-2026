# 전처리 파트

## 1. 데이터 소개


전처리 파트에서는 모델 학습에 필요한 데이터를 정제하고, 산불 발생 여부를 나타내는 타겟 변수를 생성하는 작업까지 진행.

다룬 데이터는 세 가지.

첫 번째는 공간 데이터. 강원도 전체를 100m × 100m 격자로 나누고, 각 격자에 전신주 수, 고도, 경사도, 산림 여부, 도로·하천까지의 거리 등 21개 항목을 담은 테이블. 총 273,001개 격자로 구성.

두 번째는 기상 데이터. 같은 격자 단위로 2020년부터 2025년까지 봄철(2~5월)의 기온, 습도, 풍속, 강수량 등 일별 기상 정보가 담겨 있음.

세 번째는 산불이력 데이터. 행정안전부·산림청에서 제공한 2011~2024년 전국 산불 발생 기록으로, 발생 위치 좌표, 발생 연월, 피해면적, 발생원인 등이 포함.

---

## 2. 전처리 1단계 — 데이터 정제

첫 번째로 한 작업은 공간 데이터 정제.

원본 파일에 동일한 격자가 중복으로 기록된 경우가 있었음. 확인 결과 총 31,038개의 중복 행이 발견됐고, 이를 제거해 301,305개에서 273,001개로 정리.

또한 일부 코드 컬럼이 문자열로 저장되어 있어 모델 입력에 불편했기 때문에, 임상 존재 코드, 임상 유형 코드, 임상 기원 코드, 영급 코드, 수고 코드, 갱신연도 이렇게 6개 컬럼을 숫자형으로 변환.

결과물은 `prep_master_grid.parquet`으로 저장.

---

## 3. 전처리 2단계 — 품질 분석

두 번째는 데이터 품질 점검.

결측치 분석에서는 산림 관련 컬럼들에 결측이 집중되어 있음을 확인. 이는 데이터 오류가 아니라 비산림 격자는 산림 정보가 없는 것이 당연하기 때문. 결측 구조가 명확하므로 제거가 아닌 유지 처리.

이상치 분석에서는 다섯 개 수치 컬럼을 IQR 방식으로 점검. 전신주 수는 도심이나 변전소 인근의 실제값일 가능성이 있어 그대로 유지했고, 최근접 하천 거리의 이상치 비율이 7.61%로 가장 높았음.

중복은 1단계에서 이미 처리되어 0건으로 확인.

---

## 4. 전처리 3단계 — 분포 분석

세 번째는 각 변수의 분포 특성 파악.

최근접 도로 거리의 왜도가 7.9로 가장 높았고, 최근접 하천 거리도 5.7로 높게 나왔음. 이 두 변수는 모델 학습 전 로그 변환이 필요.

반면 경사도는 왜도 0.14로 좌우 대칭에 가까워 변환 없이 사용 가능.

---

## 5. 전처리 4단계 — 상관관계 및 기상 계절성 분석

네 번째는 변수 간 관계 파악.

영급 코드와 수고 코드 사이의 상관계수가 0.53으로 가장 높게 나왔음. 두 변수를 동시에 모델에 넣을 경우 다중공선성 문제가 발생할 수 있으므로 모델링 팀에서 둘 중 하나를 선택하거나 별도 처리가 필요.

기상 데이터에서는 원본값에 10을 나눠야 실제 단위(°C, %, m/s)가 된다는 점을 검증. 계절성 분석 결과 기온은 2월에서 5월로 갈수록 상승하고, 습도는 반대로 낮아지는 패턴이 확인.

---

## 6. 전처리 5단계 — 산불 EDA 및 타겟 변수 생성

마지막은 산불 데이터 분석과 타겟 변수 생성.

분석 대상은 강원도 봄철(2~5월) 2020~2024년 산불 196건으로 필터링.

연도별로는 2022년이 49건으로 가장 많았고, 2024년이 19건으로 가장 적음. 월별로는 3월에 집중되는 경향이 확인.

타겟 변수 생성 방식은 공간 데이터는 전신주가 있는 격자만 포함하기 때문에, 산불 발생 지점과 격자를 1:1로 맞추면 매칭률이 22%에 불과. 이를 해결하기 위해 각 산불 발생 지점에서 반경 500m 이내의 모든 격자를 화재 발생 격자로 처리하는 버퍼 방식을 적용. 그 결과 매칭률이 74%로 향상.

최종적으로 273,001개 격자 중 4,137개가 화재 발생(fire=1), 268,864개가 화재 없음(fire=0)으로 표시. 비율은 약 65:1로, 화재 미발생 격자가 압도적으로 많음.

---

## 7. 최종 산출물 및 모델링 시 참고사항

전처리 결과물을 정리.

모델 입력 기준 파일은 `05_fire_label_master.parquet`. 273,001개 격자에 22개 컬럼과 타겟 변수 fire_label이 포함된 파일.

기상 데이터는 `raw/grid_date_master/` 경로에 월별로 나뉘어 저장되어 있으며, 읽을 때 fastparquet 엔진을 사용해야 함.

모델링 단계에서 주의할 점을 세 가지.

첫째, 기상 데이터 원본값은 반드시 10으로 나눈 후 사용. 그렇지 않으면 기온이 수백 도로 잡힘.

둘째, 훈련/검증 분할은 연도 기준으로 해야 함. 2020~2023년을 훈련, 2024년을 검증으로 사용.

셋째, 화재 발생 격자 비율이 1.5%로 클래스 불균형이 심함. 모델 학습 시 반드시 불균형 처리가 필요.

세부 내용은 `model_prep_point.md` 파일에 정리.
